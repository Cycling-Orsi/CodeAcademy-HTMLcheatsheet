Semantic HTML Tags

While semantic HTML tags don't provide a direct function in rendering a web page on a browser, they are an important aspect in any HTML code. When used correctly, semantic tags help compartmentalize the HTML code in a manner that provides meaning to the content it encompasses, hence the term "semantic" HTML. Because of this added meaning, semantic HTML helps with the following:

  • Accessibility: Makes webpages accessible on mobile devices and also assists screen reader tech by adding context to the code, increasing the user expereince of those with disabilities.
  • Search Engine Optimization (SEO): Allows search engines to more easily interpret and weight the content of the code, making search engines more likely to hit the website and increase user traffic to the web page.
  • Easy to Understand: Makes the source code easier for web developers to read and interpret.

Below is a table of some commonly used semantic tags.
Semantic HTML Tags
Tag Name Description
<header> Header Used to contain the introductory content of the webpage.
<nav> Navigation Used to contain the navigation content of the webpage.
<main> Main Used to contain the dominant content of the source code.
<footer> Footer Used to contain the informational content (contact info, copy right info, terms of use, etc.) at the bottom of the web page.
<section> Section Used to contain distinct parts of a webpage, such as chaptors or headings. They typically only contain one major heading element.
<article> Article Used to define the content of the webpage that makes sense on it's own. Can be all forms of media and text and can be within a section.
<aside> Aside Used to contain the content of a webpage that is not needed to understand the main point, but adds to the other elements. Examples include endnotes, pull quotes, and sidebars.
<figure> Figure Used to section off static visual media like images, illustrations, and diagrams.
<figcaption> Figure Caption Used to contain text that is directly associated with the static visual sectioned within the <figure> tag.
<audio> Audio Used to embed audio content within the webpage. It needs the src attribute to work and can use several other attributes to add user interactions.
<video> Video This element is similar in structure to the <audio> element. Used to embed video content within the webpage. It needs the src attribute to work and can use several other attributes to add user interactions.
<embed> Embed Used to embed external content at a specifc point within the webpage. This content is provided by an external application or source such as a browser plug-in. Note: browser plug-ins support has been deprecated in most modern browers, making it unwise to rely on the <embed> element. Use the above mentioned elements for their respective forms of content instead.

While all of the above tags can be used with Global Attributes, some of them either rely upon or can be enhanced with specific attributes. Below are a list of attributes that must/can be be used with the respective symantic html tag.

Semantic HTML Attributes
Attribute Name Description Associated Tags
autoplay Auto Play A Boolean attribute used to begin playing the associated media as soon as possible, even before the browser has fully downloaded the file. <audio>, <video>
controls Controls Used to display controls for the user to playback the associated media. Control options can include volume adjustment, seeking, and pause/resume. <audio>, <video>
controlslist Controls List Used to let the browser pick which control set to display whenever the browser shows its own set of controls (assuming the controls attribute has also been assigned). Values can be set to nodownload, nofullscreen, and noremoteplayback. <audio>, <video>
crossorigin Cross Origin An enumerated attribute that is used to determine whether to use CORS to retrieve the related audio file. The assigned values can either be anonymous or use-credentials. <audio>, <video>
disableremoteplayback Disable Remote Playback A Boolean attribute used to disable the remote playback option in wired and wireless technologies. <audio>, <video>
loop Loop A Boolean attribute that will automatically restart the associated media. <audio>, <video>
muted Muted A Boolean attribute that indicates whether the media will initially be silenced with a default value of false. <audio>, <video>
preload Preload An enumerated attribute that indicates to the browser how to load the media in order to give the user the best experience. The assigned values can be none, metadata, auto, or empty string. <audio>, <video>
src Source An attribute that is used to assign the URL of the media. This attribute is optional as the <source> element can also be used within the associated media block of the source code. <audio>, <video>, <embed>
disablepictureinpicture Disable Picture in Picture An attribute that prevents the browser from using a Pictire-in-Picture context menu. <video>
height Height An attribute that assigns the height of the media display area in CSS pixels. <video>, <embed>
width Width An attribute that assigns the width of the media display area in CSS pixels. <video>, <embed>
playsinline Plays Inline A Boolean attribute that forces the video "inline", ie within the element's playback area. <video>
poster Poster An attribute that assigns a URL for an image to be displayed while the video is downloading. If this isn't defined, the browser will not display anything until the first frame is downloaded, which will then be used as the poster until the rest of the video is downloaded. <video>
type Type An attribute that assigns the MIME type to use to select the plug-in to instantiate. <embed>
